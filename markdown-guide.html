Complete Markdown Guide

What is Markdown?

Markdown is a lightweight markup language created by John Gruber in 2004. It allows you to write using an easy-to-read, easy-to-write plain text format that converts to structurally valid HTML. Markdown is widely used for documentation, README files, blog posts, and anywhere you need formatted text without the complexity of HTML.

The beauty of markdown lies in its simplicity. Unlike HTML or other markup languages, markdown is designed to be readable even in its raw form. This makes it perfect for technical documentation, note-taking, and content creation where you want to focus on writing rather than formatting.

Why Use Markdown?

  • Simple and Intuitive: Easy to learn and write, even for non-technical users
  • Platform Independent: Works everywhere - GitHub, Reddit, Stack Overflow, and countless other platforms
  • Future-Proof: Plain text files that will always be readable
  • Version Control Friendly: Perfect for Git and other version control systems
  • Fast to Write: No need to take your hands off the keyboard to format
  • Portable: Convert to HTML, PDF, DOCX, and many other formats

Basic Syntax

Headers

Headers are created using hash symbols (#). The number of hashes determines the header level (H1 through H6).

Syntax:

# H1 Header
## H2 Header
### H3 Header
#### H4 Header
##### H5 Header
###### H6 Header

Result:

H1 Header

H2 Header

H3 Header

H4 Header

H5 Header
H6 Header

Best Practice: Use only one H1 per document (typically the title), and maintain a logical hierarchy. Don't skip levels (e.g., don't jump from H2 to H4).

Emphasis (Bold and Italic)

Make text bold or italic using asterisks (*) or underscores (_).

Syntax:

*Italic text* or _Italic text_
**Bold text** or __Bold text__
***Bold and italic*** or ___Bold and italic___
~~Strikethrough text~~

Result:

Italic text or Italic text

Bold text or Bold text

Bold and italic or Bold and italic

Strikethrough text

Best Practice: Be consistent - choose either asterisks or underscores and stick with it throughout your document.

Paragraphs and Line Breaks

Create paragraphs by leaving a blank line between text blocks. For a line break without starting a new paragraph, end a line with two spaces.

Syntax:

This is the first paragraph.

This is the second paragraph.

This is a line with two spaces at the end  
This appears on a new line but in the same paragraph.

Lists

Unordered Lists

Create unordered (bulleted) lists using asterisks (*), plus signs (+), or hyphens (-).

Syntax:

* Item 1
* Item 2
  * Nested item 2.1
  * Nested item 2.2
* Item 3

Result:

  • Item 1
  • Item 2
    • Nested item 2.1
    • Nested item 2.2
  • Item 3

Ordered Lists

Create ordered (numbered) lists using numbers followed by periods.

Syntax:

1. First item
2. Second item
3. Third item
   1. Nested item 3.1
   2. Nested item 3.2
4. Fourth item

Result:

  1. First item
  2. Second item
  3. Third item
    1. Nested item 3.1
    2. Nested item 3.2
  4. Fourth item

Tip: You can use "1." for all items and markdown will automatically number them correctly. This makes reordering easier.

Links

Create links using square brackets for the link text and parentheses for the URL.

Syntax:

[Link text](https://www.example.com)
[Link with title](https://www.example.com "Hover text")
[Reference-style link][1]

[1]: https://www.example.com "Reference link"

Result:

Link text

Link with title

Images

Images use the same syntax as links, but with an exclamation mark (!) in front.

Syntax:

![Alt text](image-url.jpg)
![Alt text](image-url.jpg "Image title")

Best Practice: Always include descriptive alt text for accessibility.

Blockquotes

Create blockquotes using the greater-than symbol (>).

Syntax:

> This is a blockquote.
> It can span multiple lines.
>
> > Nested blockquotes are also possible.

Result:

This is a blockquote. It can span multiple lines.

Nested blockquotes are also possible.

Horizontal Rules

Create horizontal rules (dividers) using three or more hyphens, asterisks, or underscores.

Syntax:

---
***
___

Result:

Code

Inline Code

Wrap inline code with backticks (`).

Syntax:

Use the `console.log()` function to print output.

Result:

Use the console.log() function to print output.

Code Blocks

Create code blocks using triple backticks (```) or by indenting with four spaces. Specify the language for syntax highlighting.

Syntax:

```javascript
function greet(name) {
  return `Hello, ${name}!`;
}

console.log(greet("World"));
```

Result:

function greet(name) {
  return `Hello, ${name}!`;
}

console.log(greet("World"));

Supported Languages: Markdown Viewer supports syntax highlighting for 200+ programming languages including JavaScript, Python, Java, C++, HTML, CSS, SQL, and many more.

Advanced Syntax

Tables

Create tables using pipes (|) and hyphens (-). Use colons (:) to align columns.

Syntax:

| Left Aligned | Center Aligned | Right Aligned |
|:-------------|:--------------:|--------------:|
| Row 1 Col 1  | Row 1 Col 2    | Row 1 Col 3   |
| Row 2 Col 1  | Row 2 Col 2    | Row 2 Col 3   |

Result:

Left Aligned Center Aligned Right Aligned
Row 1 Col 1 Row 1 Col 2 Row 1 Col 3
Row 2 Col 1 Row 2 Col 2 Row 2 Col 3

Tip: You don't need to make the pipes line up perfectly - markdown will still render the table correctly.

Task Lists

Create interactive checkboxes using square brackets with a space or x inside.

Syntax:

- [x] Completed task
- [ ] Incomplete task
- [ ] Another incomplete task

Result:

  • ‚òë Completed task
  • ‚òê Incomplete task
  • ‚òê Another incomplete task

Footnotes

Add footnotes using square brackets with a caret (^).

Syntax:

Here's a sentence with a footnote.[^1]

[^1]: This is the footnote content.

Definition Lists

Some markdown processors support definition lists.

Syntax:

Term 1
: Definition 1

Term 2
: Definition 2a
: Definition 2b

Best Practices

1. Use Consistent Formatting

  • Choose one style for emphasis (asterisks or underscores) and stick with it
  • Use consistent header styles throughout your document
  • Maintain consistent indentation for nested lists

2. Keep It Simple

  • Don't overuse formatting - let content speak for itself
  • Use headers to create clear document structure
  • Break long paragraphs into shorter, more readable chunks

3. Write for Readability

  • Use blank lines to separate sections
  • Keep lines under 80-100 characters when possible
  • Use lists to break up dense information
  • Add code comments in code blocks

4. Accessibility Matters

  • Always include alt text for images
  • Use descriptive link text (avoid "click here")
  • Maintain proper header hierarchy
  • Ensure sufficient contrast in any custom styling

5. Test Your Markdown

  • Preview your markdown before publishing
  • Test on different platforms if possible
  • Check that all links work correctly
  • Verify images display properly

Common Mistakes to Avoid

1. Forgetting Blank Lines

Wrong:

# Header
This paragraph won't render correctly.

Right:

# Header

This paragraph will render correctly.

2. Incorrect List Indentation

Wrong:

* Item 1
 * Nested item (only 1 space)

Right:

* Item 1
  * Nested item (2 or 4 spaces)

3. Not Escaping Special Characters

Use backslash (\) to escape special markdown characters when you want them to appear literally.

Syntax:

\* This asterisk won't create a list item
\# This won't be a header

4. Mixing HTML and Markdown

While you can use HTML in markdown, it's best to stick with markdown syntax for consistency and portability.
