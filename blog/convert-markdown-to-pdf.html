How to Convert Markdown to PDF: Complete Guide

January 25, 2025 6 min read

Converting markdown to PDF is a common need for creating professional documents, reports, and documentation. Whether you're preparing a README for print, generating documentation for clients, or creating a resume, there are multiple ways to convert markdown to PDF. This guide covers the most effective methods, from simple browser-based solutions to powerful command-line tools.

Method 1: Browser Print to PDF (Easiest)

The simplest way to convert markdown to PDF is using your web browser's built-in print function. This method requires no installation and works on any operating system.

Step-by-Step Process:

  1. Open your markdown file in a markdown viewer (like our Markdown Viewer)
  2. Ensure the content is rendered correctly with proper formatting
  3. Press Ctrl+P (Windows/Linux) or Cmd+P (Mac) to open the print dialog
  4. Select "Save as PDF" or "Microsoft Print to PDF" as the destination
  5. Adjust settings like margins, headers/footers, and page orientation
  6. Click "Save" and choose your destination folder

Pros:

  • No software installation required
  • Works on any operating system
  • Preserves styling and formatting
  • Free and instant

Cons:

  • Limited customization options
  • May have page break issues
  • Depends on browser rendering

Method 2: Pandoc (Most Powerful)

Pandoc is a universal document converter that can transform markdown into PDF with extensive customization options. It's the go-to tool for professional document conversion.

Installation:

Windows:

# Download installer from pandoc.org
# Or use Chocolatey:
choco install pandoc

Mac:

# Using Homebrew:
brew install pandoc

# You'll also need LaTeX for PDF output:
brew install basictex

Linux (Ubuntu/Debian):

sudo apt-get update
sudo apt-get install pandoc texlive-latex-base

Basic Usage:

# Simple conversion
pandoc input.md -o output.pdf

# With table of contents
pandoc input.md -o output.pdf --toc

# With custom styling
pandoc input.md -o output.pdf --css=style.css

# With metadata
pandoc input.md -o output.pdf --metadata title="My Document"

Advanced Options:

# Full-featured conversion
pandoc input.md -o output.pdf \
  --toc \
  --toc-depth=3 \
  --number-sections \
  --highlight-style=github \
  --variable=geometry:margin=1in \
  --variable=fontsize:12pt

Pros:

  • Extremely powerful and flexible
  • Professional output quality
  • Extensive customization options
  • Batch processing support
  • Free and open source

Cons:

  • Requires installation and setup
  • Steeper learning curve
  • Needs LaTeX for PDF output
  • Command-line interface

Method 3: VS Code Extensions

If you use Visual Studio Code, several extensions can convert markdown to PDF directly from your editor.

Markdown PDF Extension:

  1. Install the "Markdown PDF" extension from the VS Code marketplace
  2. Open your markdown file in VS Code
  3. Press Ctrl+Shift+P (or Cmd+Shift+P on Mac)
  4. Type "Markdown PDF: Export (pdf)" and press Enter
  5. The PDF will be saved in the same directory as your markdown file

Configuration:

You can customize the output by adding settings to your VS Code settings.json:

{
  "markdown-pdf.format": "A4",
  "markdown-pdf.margin.top": "1.5cm",
  "markdown-pdf.margin.bottom": "1cm",
  "markdown-pdf.margin.right": "1cm",
  "markdown-pdf.margin.left": "1cm",
  "markdown-pdf.displayHeaderFooter": true,
  "markdown-pdf.headerTemplate": "<div style='font-size:9px;'><span class='title'></span></div>",
  "markdown-pdf.footerTemplate": "<div style='font-size:9px;'><span class='pageNumber'></span> / <span class='totalPages'></span></div>"
}

Pros:

  • Integrated into your development workflow
  • Easy to use with keyboard shortcuts
  • Customizable settings
  • Good for regular use

Cons:

  • Requires VS Code
  • Limited to VS Code environment
  • May have rendering inconsistencies

Method 5: Node.js Tools

For developers comfortable with Node.js, there are several npm packages that can convert markdown to PDF programmatically.

markdown-pdf Package:

# Installation
npm install -g markdown-pdf

# Usage
markdown-pdf input.md -o output.pdf

# With options
markdown-pdf input.md -o output.pdf --format A4 --border 1cm

md-to-pdf Package:

# Installation
npm install -g md-to-pdf

# Usage
md-to-pdf input.md

# With custom CSS
md-to-pdf input.md --css-path style.css

Programmatic Usage:

const markdownpdf = require('markdown-pdf');
const fs = require('fs');

markdownpdf()
  .from('input.md')
  .to('output.pdf', function() {
    console.log('PDF created successfully!');
  });

Pros:

  • Programmable and automatable
  • Can be integrated into build processes
  • Batch processing support
  • Customizable with code

Cons:

  • Requires Node.js knowledge
  • Setup complexity
  • Dependency management

Tips for Better PDF Output

1. Optimize Your Markdown

  • Use proper heading hierarchy (H1 → H2 → H3)
  • Keep line lengths reasonable
  • Use tables sparingly
  • Optimize images for print (300 DPI)

2. Control Page Breaks

Add manual page breaks when needed:

<!-- HTML page break -->
<div style="page-break-before: always;"></div>

<!-- Pandoc page break -->
\newpage

3. Custom CSS for Print

@media print {
  body {
    font-family: 'Times New Roman', serif;
    font-size: 12pt;
    line-height: 1.6;
  }
  
  h1, h2, h3 {
    page-break-after: avoid;
  }
  
  table, img {
    page-break-inside: avoid;
  }
}

4. Include Metadata

Add metadata to your markdown file:

---
title: "My Document"
author: "Your Name"
date: "2025-01-25"
---

# Document Content

Troubleshooting Common Issues

Images Not Displaying

Problem: Images don't appear in the PDF.

Solution: Use absolute URLs or ensure image paths are correct relative to the markdown file.

Poor Page Breaks

Problem: Content splits awkwardly across pages.

Solution: Add manual page breaks or adjust CSS page-break properties.

Fonts Not Rendering

Problem: Custom fonts don't appear in PDF.

Solution: Use web-safe fonts or embed fonts in your CSS.

Large File Sizes

Problem: PDF file is too large.

Solution: Optimize images, use vector graphics, or compress the final PDF.

Conclusion

Converting markdown to PDF doesn't have to be complicated. For quick conversions, use the browser print method or online tools. For professional documents with custom styling, Pandoc is unbeatable. Developers will appreciate VS Code extensions for integrated workflow, while Node.js tools offer the most flexibility for automated processes.

Our Recommendations:

  • Beginners: Start with browser print method
  • Regular users: Try VS Code extensions or online tools
  • Power users: Learn Pandoc for maximum control
  • Developers: Integrate Node.js tools into your workflow
